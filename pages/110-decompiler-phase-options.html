Decompiler Phase Options

  • Disassembly and Tokenization: --asm (-a), or --asm++ (-A)
  • Parsing: --grammar (-g)
  • Parse Tree, Abstract Syntax Tree: --tree (-t), or --tree++ (-T)
  • combined short options: -AgT

$ decompyle3 -AgT /tmp/five-moved.pyc

Now that we have gone over the phases of decompilation, we give a rundown of "uncompyle6" or "decompyle3" command-line options. These will show output from each of these phases. If you use these programs, you should know about and use these when you have a problem. If you want to see more complete output for the full program that I showed in the last slides, use these options on bytecode of your choice. I use these options a lot of the time. The "dash dash asm" option or "-a" shows disassembly. To get in addition the later-produced tokenized output use: "--asm++" or "dash capital A". [DOWN] #/13/0/0 ⟶ #/13/0/1

The "dash dash grammar" option or "-g", shows grammar reductions. Output here can get very long. So unless you think there is a problem in parsing, don’t use option. Unfortunately, grammar problems occur more frequently than I would like. [DOWN] #12/0/1 ⟶ #/13/0/2

The "dash dash tree" option with short option "-t" shows the parse tree. To get the abstract syntax tree as well use "--tree++" or dash Capital T. [DOWN] #/13/0/2 ⟶ #12/0/3
